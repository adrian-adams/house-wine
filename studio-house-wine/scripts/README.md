###### Pipline Handles: ###### 

Bulk updates and additions to the follwing documents:
- Products
- Stores

######  Order of Operations ###### 

Always run in this order — products depend on store-map.json generated by the stores script:

1. upload-images (if new images added)
2. stores-to-ndjson
3. csv-to-ndjson (products)

######  Commands per script ###### 

## ./upload-images.ts (Images):

    1 → Compile
    npx tsc scripts/upload-images.ts --outDir scripts/dist --module CommonJS --target ES2020 --esModuleInterop --skipLibCheck --resolveJsonModule

    2 → Upload to Sanity
    node scripts/dist/upload-images.js

    Note: Only runs for new images — already uploaded files are skipped automatically

## ./stores-to-ndjson.ts (Stores):

    1 → Compile
    npx tsc scripts/stores-to-ndjson.ts --outDir scripts/dist --module CommonJS --target ES2020 --esModuleInterop --skipLibCheck --resolveJsonModule

    2 → Convert CSV to NDJSON
    node scripts/dist/stores-to-ndjson.js

    3 → Import to Sanity
    npx sanity dataset import data/output/stores.ndjson --dataset production --replace

## ./products-to-ndjson.ts (Products):

    1 → Compile
    npx tsc scripts/products-to-ndjson.ts --outDir scripts/dist --module CommonJS --target ES2020 --esModuleInterop --skipLibCheck --resolveJsonModule

    2 → Convert CSV to NDJSON
    node scripts/dist/products-to-ndjson.js

    3 → Import to Sanity
    npx sanity dataset import data/output/products.ndjson --dataset production --replace
    
######  Output Files (auto-generated, do not edit manually) ###### 

data/output/asset-map.json    ← filename → Sanity asset ID
data/output/store-map.json    ← storeKey → Sanity document ID  
data/output/stores.ndjson     ← Sanity-ready store documents
data/output/products.ndjson   ← Sanity-ready product documents

######  Notes ###### 

- Coordinates in stores.csv must use decimal points (.) not commas (,)
  Excel in European locales may auto-format numbers with commas — fix before running
- Store values in products.csv must exactly match storeKey values in stores.csv (case-sensitive)
- Image filenames in CSV must include file extension (e.g. gregorian-logo.svg not gregorian-logo)
